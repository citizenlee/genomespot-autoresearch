# GenomeSPOT Autoresearch

## What is this?

An autoresearch project to improve GenomeSPOT's predictive models for microbial growth conditions (oxygen tolerance, temperature, salinity, pH) from genome sequence features.

## Current baseline performance (from Barnum et al. 2024 bioRxiv)

| Condition | Model | Metric | CV Score | Test Score | N_train | N_test |
|-----------|-------|--------|----------|------------|---------|--------|
| Oxygen | LogisticRegression (20 AA features, no localization) | F1 | 0.94 | 0.97 | 2,727 | 919 |
| Temperature | Lasso (60 AA features, 3 compartments) | R² | 0.73 | 0.72 | 1,376 | 346 |
| Salinity | Lasso (60 AA features, 3 compartments) | R² | 0.81 | 0.68 | 473 | 120 |
| pH | Lasso (60 AA features, 3 compartments) | R² | 0.48 | 0.54 | 603 | 153 |

## Known weaknesses to target

1. **pH is weakest** (R²=0.48) — no strong sequence-level correlations because cytoplasmic pH is buffered
2. **Temperature below 15°C** poorly predicted (RMSE=14°C for psychrophiles)
3. **Salinity at 10-20% NaCl** poorly predicted (RMSE=4.8%)
4. Protein localization improved pH R² by +0.36 — compartment-aware features are underexploited
5. Only linear models (Lasso, LogReg) are currently used — nonlinear models untested at scale

## Project structure

```
prepare.py          # FIXED: loads data, creates CV folds, evaluates, reports metrics
train.py            # AGENT MODIFIES: model definition, feature selection, pipeline config
autoresearch.sh     # Runs prepare.py then train.py, outputs METRIC lines
autoresearch.md     # Session config (created by /autoresearch)
data/               # Training data (see setup instructions)
  training_data.tsv # Feature matrix + labels (generated by prepare_data.py)
  holdouts/         # Pre-computed train/test/CV splits (from GenomeSPOT repo)
genomespot/         # Vendored GenomeSPOT source for feature extraction
```

## Setup

1. Install dependencies: `pip install -r requirements.txt`
2. Prepare training data: `python prepare_data.py` (requires BacDive credentials or pre-computed data)
3. Verify baseline: `python prepare.py && python train.py`
4. Run autoresearch: `/autoresearch optimize composite_score`

## Key constraints

- All evaluation uses **phylogenetic holdouts at the family level** (no data leakage)
- Features must be computable from genome sequence alone (no gene annotations)
- The agent modifies ONLY `train.py`. `prepare.py` and data are fixed.
- Prefer simpler models when performance is equivalent

## Metric details

- **Primary metric**: `composite_score` = weighted mean of per-condition scores
  - Temperature R², Salinity R², pH R² (×2 weight), Oxygen F1
  - Higher is better
- **Secondary metrics**: individual R²/F1 per condition, RMSE values

## What the agent CAN modify in train.py

- Feature set selection (which feature groups, which localizations)
- Model type (linear, gradient boosting, random forest, MLP, SVM, ensembles)
- Hyperparameters (regularization, learning rate, tree depth, etc.)
- Feature selection method and number of features
- Preprocessing (scaler type, feature transformations)
- New derived features computed from existing columns
- Ensemble/stacking strategies across conditions

## What the agent must NOT modify

- `prepare.py` (data loading, splits, evaluation logic)
- Anything in `data/` (training data, holdout sets, CV folds)
- The evaluation metrics or scoring functions
- The phylogenetic CV structure
